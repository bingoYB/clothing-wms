# 服装批发进销存系统 PRD

版本：V1.0
适用业务：服装批发仓库进销存管理
适用团队规模：仓库约 8 人，含仓管、采购、老板/管理员、财务/对账人员
目标版本：一期 MVP

---

# 1. 项目背景

目前公司使用 Excel 管理库存，存在以下问题：

1. 入库、出库记录分散，容易漏记、错记。
2. 商品款号、颜色、尺码维度管理不清，导致库存数量不准确。
3. 上月盘点差异超过三万元，库存账实不符问题严重。
4. 仓管、采购、老板之间职责不清，容易出现互相修改数据、责任难追溯。
5. 采购补货缺乏库存预警依据。
6. 供应商对账主要依赖人工核对，效率低且容易出错。
7. Excel 文件多人操作时版本混乱，缺乏操作记录。

因此需要建设一套简单、高效、角色分明的进销存系统，先解决库存准确、单据留痕、采购对账和权限隔离问题。

---

# 2. 产品目标

本系统一期目标是建立一套轻量级服装批发进销存管理工具，实现：

1. 商品 SKU 标准化管理。
2. 入库、出库、盘点、库存调整全流程留痕。
3. 库存实时查询。
4. 库存预警提醒采购补货。
5. 采购单与实际入库关联。
6. 供应商对账基于实际入库数据生成。
7. 仓管、采购、老板、财务权限分明。
8. 商品信息支持扩展，不局限于固定字段。
9. 支持 Excel 导入导出，降低迁移成本。

核心原则：

**仓管管货，采购管单，老板管权限和异常；所有库存变化必须有单据、有流水、有责任人。**

---

# 3. 产品定位

本系统不是完整 ERP，也不是财务系统，而是面向服装批发业务的轻量进销存系统。

一期重点解决：

* 库存不准
* 入库出库混乱
* 商品 SKU 不规范
* 采购补货无依据
* 供应商对账困难
* 操作无记录
* 角色权限不清

一期不追求复杂审批、财务凭证、销售 CRM、智能预测等高级功能。

# 4. 用户角色

## 4.1 老板 / 管理员

老板或管理员拥有系统最高权限，负责查看全部数据、处理异常、管理用户权限。

### 核心职责

* 查看全部库存和报表。
* 管理用户账号和权限。
* 审核或确认异常单据。
* 处理库存调整。
* 查看采购金额、库存金额、供应商对账情况。

### 可操作功能

* 商品新增、编辑、停用。
* 用户角色管理。
* 入库单查看、异常处理。
* 出库单查看、异常处理。
* 采购单查看、审核、取消。
* 盘点单确认。
* 库存调整。
* 查看全部报表。
* 查看成本价和库存金额。

---

## 4.2 仓管

仓管负责实际库存操作，只处理货，不处理钱。

### 核心职责

* 入库验货。
* 确认入库。
* 创建和确认出库。
* 录入盘点数量。
* 查看库存和库存流水。
* 查看库存预警。

### 可操作功能

* 查看商品库存。
* 创建入库单。
* 确认入库。
* 创建出库单。
* 确认出库。
* 录入盘点数量。
* 查看库存流水。
* 查看库存预警。

### 不可操作或不可见

* 不可查看采购成本价。
* 不可查看供应商对账金额。
* 不可直接修改库存。
* 不可确认盘点调整。
* 不可管理用户权限。
* 不可修改采购单价格。

---

## 4.3 采购

采购负责供应商、采购单和补货，不直接改库存。

### 核心职责

* 根据库存预警创建采购单。
* 维护供应商信息。
* 查看采购入库进度。
* 查看采购报表。
* 跟进供应商到货情况。

### 可操作功能

* 查看商品资料。
* 查看库存。
* 查看库存预警。
* 创建采购单。
* 编辑采购单。
* 提交采购单。
* 取消未入库采购单。
* 查看采购入库进度。
* 维护供应商资料。
* 查看采购相关报表。

### 不可操作

* 不可确认入库。
* 不可确认出库。
* 不可直接修改库存。
* 不可做库存调整。
* 不可确认盘点结果。
* 不可管理用户权限。

---

## 4.4 财务 / 对账人员

如公司没有专职财务，可由老板兼任。

### 核心职责

* 根据采购单和实际入库数据进行供应商对账。
* 查看采购应付金额。
* 确认对账状态。
* 导出对账报表。

### 可操作功能

* 查看采购单。
* 查看入库单。
* 查看供应商对账报表。
* 确认对账状态。
* 查看库存金额。
* 导出对账数据。

### 不可操作

* 不可直接修改库存。
* 不可确认出入库。
* 不可录入盘点数量。
* 不可管理商品库存数量。

# 5. 核心业务流程

## 5.1 商品建档流程

### 流程说明

商品资料是所有库存、采购、入库、出库的基础。服装行业必须支持款号、颜色、尺码维度，同时商品信息必须可扩展。

### 流程

1. 管理员或有权限人员创建商品。
2. 填写商品基础信息。
3. 配置颜色、尺码，生成 SKU。
4. 设置库存预警值。
5. 如有需要，填写扩展属性。
6. 商品启用后，可用于采购、入库、出库和库存查询。

### 关键规则

* 商品编号必须唯一。
* SKU 编码必须唯一。
* 默认 SKU 规则为：款号 + 颜色 + 尺码。
* 商品停用后，不允许新增采购或出入库，但历史记录保留。
* 商品扩展字段可由管理员配置。

---

## 5.2 采购流程

### 流程说明

采购负责创建采购单，但采购单本身不增加库存。只有仓管实际确认入库后，库存才增加。

### 流程

1. 采购查看库存预警。
2. 采购创建采购单。
3. 采购选择供应商和商品 SKU。
4. 填写采购数量、采购单价、预计到货日期。
5. 采购提交采购单。
6. 仓管根据采购单收货。
7. 入库完成后，采购单状态更新为部分入库或已完成。

### 采购单状态

| 状态   | 说明             |
| ---- | -------------- |
| 草稿   | 采购正在编辑，未提交     |
| 已提交  | 采购单已提交，可用于到货入库 |
| 部分入库 | 部分商品已实际入库      |
| 已完成  | 采购单商品全部入库      |
| 已取消  | 采购单作废          |

### 关键规则

* 采购单不直接影响库存。
* 采购单可以关联多个入库单。
* 实际入库数量可能小于、等于或大于采购数量。
* 采购数量和实收数量不一致时，必须记录差异。
* 已有入库记录的采购单不可直接删除。

---

## 5.3 入库流程

### 流程说明

入库用于记录实际到货并增加库存。入库可来源于采购到货、退货入库、调整入库等。

### 标准流程

1. 仓管选择采购单或创建无采购来源入库单。
2. 系统带出采购单明细。
3. 仓管录入实际收到数量。
4. 系统计算采购数量与实收数量差异。
5. 仓管填写差异原因。
6. 仓管确认入库。
7. 系统增加库存。
8. 系统生成库存流水。
9. 采购单入库进度更新。

### 入库类型

| 类型   | 说明      |
| ---- | ------- |
| 采购入库 | 供应商采购到货 |
| 退货入库 | 客户退货入库  |
| 调整入库 | 管理员调整库存 |
| 其他入库 | 特殊情况    |

### 关键规则

* 入库确认前不影响库存。
* 入库确认后库存增加。
* 已确认入库单不可普通删除。
* 已确认入库单如需修改，需走冲销或管理员调整。
* 每次入库必须生成库存流水。
* 采购数量和实收数量不一致时，必须显示差异。

---

## 5.4 出库流程

### 流程说明

出库用于记录销售发货、调货、报损等库存减少行为。

### 标准流程

1. 仓管创建出库单。
2. 选择客户或出库原因。
3. 添加商品 SKU 和出库数量。
4. 系统校验库存是否足够。
5. 仓管确认出库。
6. 系统减少库存。
7. 系统生成库存流水。

### 出库类型

| 类型   | 说明           |
| ---- | ------------ |
| 销售出库 | 批发销售发货       |
| 调拨出库 | 后续多仓使用，一期可预留 |
| 报损出库 | 破损、丢失等       |
| 其他出库 | 特殊情况         |

### 关键规则

* 库存不足时，不允许确认出库。
* 出库确认后库存减少。
* 已确认出库单不能普通删除。
* 出库单修改需通过冲销或管理员调整。
* 出库必须生成库存流水。

---

## 5.5 库存查询流程

### 流程说明

库存查询是仓管、采购和老板的高频功能。不同角色看到的字段不同。

### 查询维度

支持按以下条件查询：

* 款号
* 商品名称
* SKU 编码
* 颜色
* 尺码
* 分类
* 供应商
* 库存状态
* 是否低库存
* 自定义商品属性
* 条码，二期可用

### 库存字段

| 字段     | 说明         |
| ------ | ---------- |
| 商品编号   | 系统商品编号     |
| 款号     | 服装款号       |
| 商品名称   | 商品名称       |
| 颜色     | SKU 维度     |
| 尺码     | SKU 维度     |
| 当前库存   | 系统账面库存     |
| 预警值    | 低库存判断依据    |
| 最近入库时间 | 最近一次入库     |
| 最近出库时间 | 最近一次出库     |
| 成本价    | 采购、老板、财务可见 |
| 库存金额   | 老板、财务可见    |
| 扩展属性   | 根据配置显示     |

---

## 5.6 库存预警流程

### 流程说明

库存预警用于提醒采购补货。

### 预警规则

当 SKU 当前库存 ≤ SKU 预警值时，进入库存预警列表。

### 预警字段

| 字段     | 说明     |
| ------ | ------ |
| 款号     | 商品款号   |
| 商品名称   | 商品名称   |
| 颜色     | SKU 颜色 |
| 尺码     | SKU 尺码 |
| 当前库存   | 当前库存数量 |
| 预警值    | 最低库存   |
| 建议采购数量 | 可选     |
| 默认供应商  | 补货参考   |
| 最近入库时间 | 辅助判断   |
| 最近出库时间 | 辅助判断   |

### 一期规则

一期只做简单预警：

**当前库存 ≤ 预警值，即触发预警。**

不做复杂销量预测、季节预测和智能补货模型。

---

## 5.7 盘点流程

### 流程说明

盘点用于核对实际库存和系统库存差异。盘点结果必须经过管理员确认后才能影响库存。

### 流程

1. 管理员或仓管创建盘点单。
2. 选择盘点范围。
3. 仓管录入实盘数量。
4. 系统计算账面库存与实盘库存差异。
5. 仓管提交盘点结果。
6. 管理员确认盘点差异。
7. 系统按差异调整库存。
8. 系统生成库存流水。
9. 形成库存差异报表。

### 盘点范围

| 范围     | 说明         |
| ------ | ---------- |
| 全仓盘点   | 盘点全部商品     |
| 分类盘点   | 按商品分类盘点    |
| 款号盘点   | 按指定款号盘点    |
| SKU 盘点 | 按指定 SKU 盘点 |

### 关键规则

* 仓管可录入实盘数量。
* 仓管不可自行确认库存调整。
* 管理员确认后，库存才变化。
* 盘点差异必须记录原因。
* 盘点调整必须生成库存流水。

---

## 5.8 供应商对账流程

### 流程说明

供应商对账基于采购单和实际入库数量生成。应付金额按实际入库数量计算。

### 流程

1. 财务选择供应商和对账周期。
2. 系统查询该周期内已确认入库记录。
3. 系统关联采购单价格。
4. 系统计算应付金额。
5. 财务核对数量、单价和差异。
6. 财务确认对账。
7. 系统标记对账状态。
8. 支持导出对账表。

### 对账规则

* 应付金额 = 实际入库数量 × 采购单价。
* 对账以入库单为准，不以采购单数量为准。
* 采购数量和实收数量不一致时，必须显示差异。
* 已对账记录不可随意修改。
* 如需修改，需管理员调整并记录原因。

# 6. 功能需求

## 6.1 用户与权限管理

### 功能说明

系统需要支持多角色权限，避免不同岗位看到或修改不该操作的数据。

### 功能点

| 功能   | 说明               |
| ---- | ---------------- |
| 用户登录 | 用户通过账号密码登录       |
| 用户管理 | 管理员新增、停用用户       |
| 角色管理 | 设置老板、仓管、采购、财务等角色 |
| 权限控制 | 控制菜单、按钮、字段可见性    |
| 操作记录 | 记录关键操作人和时间       |

### 一期角色

* 老板 / 管理员
* 仓管
* 采购
* 财务 / 对账人员

---

## 6.2 商品管理

### 功能说明

商品管理用于维护商品基础资料、SKU 和扩展属性。

### 商品核心字段

| 字段    | 说明           | 是否必填 |
| ----- | ------------ | ---- |
| 商品编号  | 系统唯一编号，可自动生成 | 是    |
| 商品名称  | 商品名称         | 是    |
| 款号    | 服装业务核心字段     | 是    |
| 分类    | 上衣、裤子、外套、裙装等 | 否    |
| 单位    | 件、包、箱        | 是    |
| 默认供应商 | 采购补货参考       | 否    |
| 成本价   | 采购成本         | 否    |
| 批发价   | 销售参考价        | 否    |
| 商品图片  | 商品主图         | 否    |
| 状态    | 启用 / 停用      | 是    |
| 备注    | 补充说明         | 否    |

### SKU 字段

| 字段     | 说明      | 是否必填 |
| ------ | ------- | ---- |
| SKU 编码 | 唯一编码    | 是    |
| 商品 ID  | 所属商品    | 是    |
| 颜色     | 颜色维度    | 是    |
| 尺码     | 尺码维度    | 是    |
| 条码     | 二期扫码使用  | 否    |
| 预警值    | 低库存提醒   | 否    |
| 状态     | 启用 / 停用 | 是    |

### 默认 SKU 规则

默认以以下规则生成 SKU：

**款号 + 颜色 + 尺码 = SKU**

示例：

| 款号    | 颜色 | 尺码 | SKU        |
| ----- | -- | -- | ---------- |
| A1023 | 黑色 | M  | A1023-黑色-M |
| A1023 | 黑色 | L  | A1023-黑色-L |
| A1023 | 白色 | M  | A1023-白色-M |

---

## 6.3 商品信息扩展

### 功能说明

商品信息不得局限于固定字段。系统应支持管理员配置自定义商品属性，以适应后续业务变化。

### 设计原则

商品信息分为三层：

1. 系统核心字段。
2. 业务常用字段。
3. 自定义扩展字段。

### 第一层：系统核心字段

这些字段固定存在，影响业务流程，不允许删除：

* 商品编号
* 商品名称
* 款号
* SKU 编码
* 颜色
* 尺码
* 单位
* 状态

### 第二层：业务常用字段

这些字段默认提供，可配置是否启用：

* 分类
* 默认供应商
* 成本价
* 批发价
* 库存预警值
* 商品图片
* 备注

### 第三层：自定义扩展字段

管理员可以根据业务增加字段，例如：

* 季节
* 年份
* 面料
* 风格
* 版型
* 厚薄
* 适用人群
* 货源档口
* 执行标准
* 洗涤说明
* 内部标签
* 直播间编号
* 备注说明

### 自定义属性配置项

| 配置项    | 说明                   |
| ------ | -------------------- |
| 属性名称   | 例如：面料、季节、版型          |
| 属性类型   | 文本、数字、单选、多选、日期、图片、附件 |
| 是否必填   | 建档时是否必须填写            |
| 是否启用   | 不使用时可停用              |
| 是否用于筛选 | 是否在商品列表中作为筛选条件       |
| 是否用于搜索 | 是否支持关键词搜索            |
| 是否列表显示 | 是否显示在商品列表            |
| 适用分类   | 例如只对外套显示厚薄           |
| 排序     | 控制字段展示顺序             |

### 扩展字段应用范围

## 6.9 库存流水

### 功能说明

库存流水记录每一次库存变化，是库存追溯和责任划分的核心。

### 字段

| 字段    | 说明              |
| ----- | --------------- |
| 流水编号  | 系统自动生成          |
| SKU   | 商品 SKU          |
| 变化类型  | 入库、出库、盘点调整、库存调整 |
| 变化前库存 | 操作前库存           |
| 变化数量  | 正数或负数           |
| 变化后库存 | 操作后库存           |
| 来源单据  | 入库单、出库单、盘点单、调整单 |
| 操作人   | 自动记录            |
| 操作时间  | 自动记录            |
| 备注    | 可选              |

### 关键规则

* 库存流水不可删除。
* 库存流水不可修改。
* 任意 SKU 当前库存必须能追溯到历史流水。
* 每一次库存变化必须生成流水。

# 10. 关键业务规则

## 10.1 库存不能直接随便改

库存只能由以下操作改变：

* 确认入库。
* 确认出库。
* 盘点确认。
* 管理员库存调整。

普通用户不能直接修改库存数量。

---

## 10.2 所有库存变化必须留痕

每一次库存变化都必须记录：

* 操作人
* 操作时间
* SKU
* 变化前库存
* 变化数量
* 变化后库存
* 来源单据
* 备注

---

## 10.3 已确认单据不能普通删除

已确认的入库单、出库单、盘点单不能直接删除。

如需修正，必须通过：

* 冲销
* 反向单据
* 管理员调整

---

## 10.4 采购和入库必须分开

采购单只代表采购计划，不代表实际库存增加。

只有仓管确认入库后，库存才增加。

---

## 10.5 出库不能超过库存

确认出库时，系统必须校验库存。

如果库存不足，不允许确认出库。

---

## 10.6 对账按实际入库数量计算

供应商应付金额按实际入库数量计算，不按采购单数量计算。

公式：

**应付金额 = 实收数量 × 采购单价**

---

## 10.7 盘点调整必须经管理员确认

仓管可以录入实盘数，但不能自行调整系统库存。

盘点差异必须由管理员确认后才更新库存。

---

## 10.8 商品扩展字段不能影响核心库存逻辑

商品自定义字段用于描述、筛选、搜索和报表展示，但库存计算仍以 SKU 为核心。

# 13. 验收标准

系统上线前需满足以下验收标准：

1. 任意 SKU 当前库存可以追溯到所有入库、出库、盘点调整流水。
2. 仓管不能查看成本价和供应商对账金额。
3. 采购不能直接修改库存。
4. 采购单不影响库存，只有确认入库后库存增加。
5. 出库数量超过库存时，系统禁止确认出库。
6. 入库数量与采购数量不一致时，系统显示差异并要求填写原因。
7. 盘点差异必须经过管理员确认后才影响库存。
8. 所有库存变化都必须生成库存流水。
9. 已确认入库单、出库单、盘点单不能被普通用户删除。
10. 库存预警能按 SKU 触发。
11. 商品资料支持自定义扩展字段。
12. 自定义字段支持商品列表展示、筛选、导入和导出。
13. 报表支持按日期、供应商、款号、SKU 筛选。
14. 当前库存、入库明细、出库明细、对账表、盘点差异表均可导出 Excel。
15. 用户权限与角色矩阵一致。

# 15. 一期成功标准

一期上线后，应达到以下效果：

1. 库存不再依赖多人维护 Excel。
2. 入库、出库、盘点都有单据记录。
3. 每次库存变化都有责任人和时间。
4. 仓管、采购、财务权限清楚。
5. 采购可根据库存预警补货。
6. 老板能快速看到库存差异和对账数据。
7. 供应商对账基于实际入库，不再靠人工反复核对。
8. 商品字段可以根据业务变化继续扩展。

---

# 16. 总结

本系统一期应围绕以下业务闭环建设：

**商品建档 → 采购下单 → 仓库入库 → 库存增加 → 销售出库 → 库存减少 → 库存预警 → 再采购 → 盘点调整 → 供应商对账**

核心设计思想：

**库存管理字段要稳定，商品描述字段要可扩展；业务流程要简单，库存变化要严格。**
